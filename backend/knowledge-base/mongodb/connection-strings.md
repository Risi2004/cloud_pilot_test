# Connect to an Atlas Cluster[](https://www.mongodb.com/docs/atlas/connect-to-database-deployment/#connect-to-an-atlas-cluster "Permalink to this heading")

Copy page

This page describes how to connect to your Atlas cluster using the Atlas CLI or the Atlas UI.

## Prerequisites[](https://www.mongodb.com/docs/atlas/connect-to-database-deployment/#prerequisites "Permalink to this heading")

#### IP Access List[](https://www.mongodb.com/docs/atlas/connect-to-database-deployment/#ip-access-list "Permalink to this heading")

To access a cluster, you must connect from an IP address on the Atlas project's IP access list. If you need to add an IP address to the IP access list, you can do so in the Connect dialog box. You can also add the IP address from the [Network Access tab.](https://www.mongodb.com/docs/atlas/security/ip-access-list/#std-label-add-to-access-list/)

#### Database User[](https://www.mongodb.com/docs/atlas/connect-to-database-deployment/#database-user "Permalink to this heading")

To access a cluster, you must create a [database user](https://www.mongodb.com/docs/manual/core/security-users/) with access to the desired databases on your Atlas cluster. Database users are separate from Atlas users. Database users have access to MongoDB databases, while Atlas users have access to the Atlas application itself.

You can create a database user to access your Atlas cluster in the Connect dialog box. You can also add the database user from the [Cluster view.](https://www.mongodb.com/docs/atlas/security-add-mongodb-users/#std-label-mongodb-users/)

### Open Ports 27015 to 27017 to Access Atlas Databases[](https://www.mongodb.com/docs/atlas/connect-to-database-deployment/#open-ports-27015-to-27017-to-access-atlas-databases "Permalink to this heading")

Make sure your application can reach your MongoDB Atlas environment. To add the inbound network access from your application environment to Atlas, do one of the following:

1.   Add the public IP addresses to your IP access list 
2.   Use [VPC / VNet peering](https://www.mongodb.com/docs/atlas/security-vpc-peering/#std-label-vpc-peering/) to add private IP addresses. 
3.   Add [private endpoints.](https://www.mongodb.com/docs/atlas/security-configure-private-endpoints/#std-label-private-endpoint/) 

## Tip

[IP Access List](https://www.mongodb.com/docs/atlas/setup-cluster-security/#std-label-security-ip-access-list/)

If your firewall blocks outbound network connections, you must also open outbound access from your application environment to Atlas. You must configure your firewall to allow your applications to make outbound connections to ports 27015 to 27017 to TCP traffic on the hostnames or IP addresses of your cluster. This grants your applications access to databases stored on Atlas.

To obtain the current list of cluster IP addresses, use the [Return All IP Addresses for One Project endpoint](https://www.mongodb.com/docs/api/doc/atlas-admin-api-v2/operation/operation-getgroupipaddresses) of the Atlas Administration API.

`curl --header "Authorization: Bearer <access-token>" \     --header "Accept: application/vnd.atlas.2025-03-12+json" \     --request GET \     "https://cloud.mongodb.com/api/atlas/v2/groups/{GROUP-ID}/ipAddresses"`

## Tip

[Add Project Access from a Project](https://www.mongodb.com/docs/atlas/configure-api-access/#std-label-create-project-api-key/)

## Note

By default, MongoDB Atlas clusters do not need to be able to initiate connections to your application environments. If you wish to enable Atlas clusters with [LDAP authentication and authorization](https://www.mongodb.com/docs/atlas/security-ldaps/#std-label-ldaps-authentication-authorization/), you must allow network access from Atlas clusters directly to your secure LDAP. You can allow access to your LDAP by using public or private IPs as long as a public DNS hostname points to an IP that the Atlas clusters can access.

If you are not using [VPC / VNet peering](https://www.mongodb.com/docs/atlas/security-vpc-peering/#std-label-vpc-peering/) and plan to connect to Atlas using public IP addresses, see the following pages for additional information:

*   [Can I specify my own VPC for my MongoDB Atlas project?](https://www.mongodb.com/docs/atlas/reference/faq/networking/#std-label-faq-atlas-side-hostnames/) 
*   [Do Atlas cluster's public IPs ever change?](https://www.mongodb.com/docs/atlas/reference/faq/networking/#std-label-faq-public-ip-changes/) 

## Considerations[](https://www.mongodb.com/docs/atlas/connect-to-database-deployment/#considerations "Permalink to this heading")

Atlas does not guarantee that host names remain consistent with respect to node types during topology changes.

## Example

If you have a cluster named `foo123` containing an analytics node `foo123-shard-00-03-a1b2c.mongodb.net:27017`, Atlas does not guarantee that specific host name will continue to refer to an analytics node after a topology change, such as [scaling a cluster](https://www.mongodb.com/docs/atlas/scale-cluster/) to modify its number of nodes or regions.

## Connect to Your Cloud Cluster[](https://www.mongodb.com/docs/atlas/connect-to-database-deployment/#connect-to-your-cloud-cluster "Permalink to this heading")

To get your deployment's connection string, you can use the Atlas CLI or Atlas UI:

Interface 

Atlas CLI

You can use the Atlas CLI to get your deployment's standard connection string.

To return the SRV connection strings for your Atlas cluster using the Atlas CLI, run the following command:

`atlas clusters connectionStrings describe <clusterName> [options]`

To learn more about the command syntax and parameters, see the Atlas CLI documentation for [atlas clusters connectionStrings describe.](https://www.mongodb.com/docs/atlas/cli/current/command/atlas-clusters-connectionStrings-describe/)

For example, if the Atlas CLI returns the following connection string:

`mongodb+srv://mycluster.abcd1.mongodb.net`

You can connect to your deployment using [`mongosh`](https://www.mongodb.com/docs/mongodb-shell/#mongodb-binary-bin.mongosh) by including the connection string in the [`mongosh`](https://www.mongodb.com/docs/mongodb-shell/#mongodb-binary-bin.mongosh) command:

`mongosh "mongodb+srv://mycluster.abcd1.mongodb.net/myFirstDatabase" --apiVersion 1 --username <username>`

## Note

To successfully connect to Atlas, you must add your username and database name to the connection string. In some cases, such as when you're using [`mongosh`](https://www.mongodb.com/docs/mongodb-shell/#mongodb-binary-bin.mongosh) from the terminal, Atlas prompts you to enter the password for the database user.

### Connect to a Local Deployment with the Atlas CLI[](https://www.mongodb.com/docs/atlas/connect-to-database-deployment/#connect-to-a-local-deployment-with-the-atlas-cli "Permalink to this heading")

To connect to an Atlas deployment using the Atlas CLI, run the following command:

`atlas deployments connect [deploymentName] [options]`

To learn more about the command syntax and parameters, see the Atlas CLI documentation for [atlas deployments connect.](https://www.mongodb.com/docs/atlas/cli/current/command/atlas-deployments-connect/)

## Connect to Your Cloud Cluster Using Other Methods[](https://www.mongodb.com/docs/atlas/connect-to-database-deployment/#connect-to-your-cloud-cluster-using-other-methods "Permalink to this heading")

In order to connect to your cluster, you must get your deployment's [Connection Strings](https://www.mongodb.com/docs/manual/reference/connection-string/#std-label-find-connection-string). Once you have the connection string, you can connect to your deployment by using the following connection methods:

*   [Connect to a Cluster via Client Libraries](https://www.mongodb.com/docs/atlas/driver-connection/#std-label-connect-via-driver/) 
*   [Connect to a Cluster via Compass](https://www.mongodb.com/docs/atlas/compass-connection/#std-label-atlas-connect-via-compass/) 
*   [Connect to a Cluster via mongosh](https://www.mongodb.com/docs/atlas/mongo-shell-connection/#std-label-connect-mongo-shell/) 
*   [Connect to a Cluster via VS Code](https://www.mongodb.com/docs/atlas/mongodb-for-vscode/#std-label-mongodb-for-vscode/) 
*   [Connect to a Cluster via the BI Connector](https://www.mongodb.com/docs/atlas/bi-connection/) 

## Note

To connect using [`mongodump`](https://www.mongodb.com/docs/database-tools/mongodump/#mongodb-binary-bin.mongodump) or [`mongorestore`](https://www.mongodb.com/docs/database-tools/mongorestore/#mongodb-binary-bin.mongorestore), use the [Command Line Tools](https://www.mongodb.com/docs/atlas/command-line-tools/) option in the Cluster sidebar. This creates an auto-generated template for connecting to your Atlas cluster with your preferred tool.

If you are experiencing issues connecting to your cluster, see [Troubleshoot Connection Issues.](https://www.mongodb.com/docs/atlas/troubleshoot-connection/)
