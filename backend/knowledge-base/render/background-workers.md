# Background Workers

## Offload asynchronous tasks to a separate service listening on a queue.

**Looking to run high-volume distributed tasks?**

Currently in beta, [Render Workflows](https://render.com/docs/workflows) provides an all-in-one worker model with managed queuing, automatic retries, and rapid spin-up.

[Try Workflows](https://render.com/docs/workflows)

**Background workers** are services that run continuously (like a [web service](https://render.com/docs/web-services) or a [private service](https://render.com/docs/private-services)), but they don't receive any incoming network traffic. Instead, these services usually poll a task queue (such as one backed by a [Render Key Value](https://render.com/docs/key-value) instance) and process new tasks as they come in:

![Image 1: Diagram of a background worker polling a task queue](https://render.com/docs-assets/80bc4746f7424ebcf0e990da494ec6a3b4f43458f8ac6b0f2c351ac976bfa21f/background-worker.png)

Background workers help to keep your apps responsive by offloading long-running, asynchronous tasks from your services in the critical request path.

Common worker tasks include:

*   Processing media files
*   Generating reports
*   Interacting with third-party APIs, such as Stripe, Twilio, or AI models

## [](https://render.com/docs/background-workers#popular-worker-frameworks)Popular worker frameworks

You can use the frameworks below to simplify polling a task queue backed by a Redis®-like store (such as [Render Key Value](https://render.com/docs/key-value)).

| Language | Framework |
| --- | --- |
| **Python** | Celery ([see quickstart](https://render.com/docs/deploy-celery)) |
| **Ruby** | Sidekiq ([see quickstart](https://render.com/docs/deploy-rails-sidekiq)) |
| **Node.js** | [BullMQ](https://bullmq.io/) |
| **Go** | [Asynq](https://github.com/hibiken/asynq) |
| **Elixir** | [Oban](https://hexdocs.pm/oban/Oban.html) **Note:** This framework integrates with Render Postgres instead of Key Value. |
| **Rust** | [apalis](https://github.com/geofmureithi/apalis) |
